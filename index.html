DEVELOPMENT PROCESS

ALPHA VERSION
DESCRIPTION

Initial prototype with core "Risk Meter" mechanics and 8 panels. Established the dark neon aesthetic.

FEEDBACK (Sponsor)

Sponsor suggested focusing more on visual narrative and adding interactive elements to engage users.

IMPROVED

Refined the Risk Meter logic and reduced panel gaps for a better "Webtoon" feel.

BETA VERSION
DESCRIPTION

Expanded to 12 panels. Introduced "AI Mode" prototype and improved scroll-tracking.

FEEDBACK (User Testing)

Users mentioned they wanted to know how far they were in the story. Some lag on mobile.

IMPROVED

Added a Progress Bar and Panel Counter (1/16). Optimized code for faster loading.

FINAL VERSION
DESCRIPTION

Full 16-panel interactive story with stable AI Mode, Action Plan, and high-performance scroll logic.

IMPROVED

Final image optimization using TinyPNG. Implemented responsive CSS for all screen sizes.

PROJECT REFLECTION

What improved the most from Alpha to Final?

The **narrative flow and technical stability** saw the most growth. We transitioned from a basic 8-panel scroll to a complex 16-panel story with a fully functional AI-Mode that visualizes the psychological state of the character.

What feedback was most useful?

The feedback regarding **user orientation** was vital. Adding the Progress Bar and Panel Counter significantly improved the usability of the site, making the long-scroll format much more user-friendly.

What would you still improve?

I would implement **parallax CSS animations** to create a deeper sense of immersion and investigate advanced AI training to maintain a more consistent art style across all hallucination scenes.
